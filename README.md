# 🍽️ Restaurant Management System

> A coursework project for the **"Network Applications"** course — a Spring Boot backend system for managing restaurant operations between clients and administrators.

---

## 📖 Description

**Restaurant** is a server-side web application that models the full lifecycle of a restaurant order:

1. A **Client** browses the **Menu** and places an **Order**.
2. An **Administrator** reviews and confirms the order, then sends it to the kitchen.
3. Once the order is ready, the **Administrator** issues an **Invoice**.
4. The **Client** pays the invoice to complete the transaction.

The system is built with **Java** and **Spring Boot**, backed by a relational database defined in `cursework.sql`.

## 🚀 Getting Started

### Prerequisites

- **JDK 17+** (or the version specified in `pom.xml`)
- **MySQL** or **PostgreSQL** running locally
- **Maven** (or use the included `mvnw` wrapper)

### 1. Set up the database

Import the provided SQL schema:

```bash
# MySQL
mysql -u root -p your_database < cursework.sql

# PostgreSQL
psql -U postgres -d your_database -f cursework.sql
```

### 2. Configure the application

Edit `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/your_database
spring.datasource.username=your_username
spring.datasource.password=your_password
```

### 3. Build and run

```bash
# Linux / macOS
./mvnw spring-boot:run

# Windows
mvnw.cmd spring-boot:run
```

The application will start at `http://localhost:8080` by default.

## 📋 Core Domain Entities

| Entity    | Description                                  |
|-----------|----------------------------------------------|
| `Client`  | A restaurant customer who places orders      |
| `Menu`    | List of available dishes and their prices    |
| `Order`   | A client's selection of menu items           |
| `Invoice` | A bill generated by the admin for an order   |
